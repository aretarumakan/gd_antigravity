# GoogleAntigravity

スクールエージェントチャットスペース内のGemini DIVE 「GoogleAntigravity」プロジェクトのアーカイブ兼作業場です。

## 🚀 Mission

**「作ったものや具体案、できること等をストックして、いつでも見返せる場所にする」**

このプロジェクトは、GeminiDiveの活動の中で生まれた成果物やアイデアを蓄積し、インパクトのあるアウトプットを生み出し続けるための基盤です。Facebookのタイムラインのように流れてしまわないよう、ここでしっかりと管理・ブラッシュアップしていきます。

## 📂 Directory Structure

このレポジトリは、参加者が迷わず自由に投稿・閲覧できるよう、以下の構成になっています。

- **`projects/`**: 具体的に動き出したプロジェクトのフォルダ。
    - プロジェクトごとにサブフォルダを作成してファイルを格納します。
- **`ideas/`**: まだ形になっていないアイデア、メモ、提案など。
    - 気軽にMarkdownファイルなどで追加してください。
- **`resources/`**: 共有の資料、リンク集、便利なテンプレートなど。
- **`README.md`**: このファイル。プロジェクトの全体像。
- **`CONTRIBUTING.md`**: 参加方法、貢献のガイドライン。

## 🤝 How to Participate

このプロジェクトへの参加は、**プルリクエスト（Pull Request）** ベースで行います。
詳しい手順は [CONTRIBUTING.md](./CONTRIBUTING.md) をご覧ください。

Gitに慣れていない方でも参加しやすいよう、手順をまとめています。
まずは `ideas/` フォルダに自分のアイデアを追加してみることから始めてみましょう！

## 🚨 重要事項

データの取り扱い監視やAI学習への利用に関する重要な警告については、[データの取り扱いに関する警告 (重要)](./警告.md) を必ず確認してください。

## ⚠️ 注意事項（必ずお読みください）

プルリクエストを送る前に、以下の点を必ず確認してください。

### 🔐 絶対に含めてはいけないもの

| 種類 | 例 | なぜダメ？ |
| :--- | :--- | :--- |
| **APIキー・シークレット** | `sk-xxxx...`, `OPENAI_API_KEY=...` | 悪意ある人に悪用され、高額請求される恐れがあります |
| **パスワード** | ログイン情報、データベースのパスワード | アカウントを乗っ取られる危険があります |
| **個人情報** | 本名、住所、電話番号、メールアドレス | プライバシーの侵害、詐欺に使われる恐れがあります |
| **認証トークン** | アクセストークン、リフレッシュトークン | あなたになりすましてサービスを使われてしまいます |
| **秘密鍵・証明書** | `.pem`, `.key` ファイル、SSH鍵 | サーバーやアカウントへの不正アクセスを許してしまいます |

### ✅ 投稿前チェックリスト

- [ ] コードに API キーやパスワードがハードコーディングされていないか？
- [ ] 設定ファイル（`.env` など）が含まれていないか？
- [ ] スクリーンショットに個人情報が映り込んでいないか？
- [ ] サンプルデータに実在の人物情報が含まれていないか？

### 💡 もし間違えて公開してしまったら

**すぐに以下の対応をしてください：**

1. **該当のキーやパスワードを無効化・変更する**（これが最優先！）
2. プルリクエストをクローズする
3. 必要に応じてメンバーに報告する

> ⚠️ GitHubの履歴には残り続けるため、一度公開した情報は「なかったこと」にはできません。**公開前の確認が最も重要です。**

## 📚 Git用語集

「Git（ギット）」は、みんなで一つの作品を作るための魔法のノートです。

| 用語 | イメージ | 説明 |
| :--- | :--- | :--- |
| **レポジトリ**<br>(Repository) | 🏫 **教室の本棚** | みんなのファイルが保管されている場所。「プロジェクトそのもの」です。 |
| **クローン**<br>(Clone) | 🎒 **自分の机に持っていく** | 本棚からノートを持ってきて、自分の手元（パソコン）にコピーすること。 |
| **ブランチ**<br>(Branch) | 📖 **コピーしたノート** | 本物のノートにいきなり書くと失敗するかも...💦<br>だから「コピー」をとって、そこで自由に下書きをします。この「下書き用コピー」のことです。 |
| **コミット**<br>(Commit) | 💾 **セーブ** | ゲームのセーブポイント！<br>「ここまで書いたよ」と記録を残すこと。失敗しても、いつでもこの場所に戻れます。 |
| **プッシュ**<br>(Push) | 📮 **提出** | 手元で書いたノートを、インターネット上の本棚（GitHub）に送ること。 |
| **プルリクエスト**<br>(Pull Request) | 🙋 **先生、見てください！** | 「下書き（ブランチ）ができたので、本物のノート（main）に書き写していいですか？」とお願いすること。<br>略して「プルリク」「PR」とも呼びます。 |
| **マージ**<br>(Merge) | ✨ **合体！** | OKが出たら、下書きの内容を本物のノートに取り込みます。これでみんなが見れるようになります！ |
